## 一、概念题 🟢

## 三、架构 / 陷阱题 🔴

## 参考答案要点

<details>
<summary>点开核对</summary>

- **1**：把海量小文件打进单个大文件自管理；OS 存小文件会拖慢 IO、浪费簇空间、难加密校验。
- **2**：HeaderData + BlockData[] + StringData[]（元数据区）+ 簇数据区。
- **3**：4KB；分配粒度（文件按簇对齐）。
- **4**：`GFF`；版本、随机加密字节、MaxFileCount、MaxBlockCount、BlockCount。
- **6**：先查空闲块表找 ≥ 所需长度的最佳匹配块复用；无则在数据区末尾新增块、BlockCount++。
- **7**：按长度分组支持"最佳匹配"——找最接近所需大小的空闲块，减少浪费与碎片。
- **8**：不收缩；块标记空闲(StringIndex=-1)，进空闲表待复用。
- **9**：StringData 用头里随机 encryptBytes 做自异或(XOR)，定长 255 字节。
- **10**：只改块的 StringData(文件名)与文件名→块映射；簇数据不动。
- **11**：1 簇 = 4096 字节（向上取整）。
- **12**：换来删除 O(1)、避免重排整个归档；代价是空间碎片，物理文件不会因删除变小。
- **13**：都是"复用而非释放"——ObjectPool 复用业务对象、ReferencePool 复用 IReference、FileSystem 复用磁盘块/簇。区别在复用的资源（内存对象 vs 磁盘空间）。
- **14**：写时分配块→登记块表→写簇数据→存文件名(名表)→m_FileDatas 映射；漏改任一即元数据与数据不一致，读取错乱/损坏。
- **15**：定长预留让"按索引计算偏移定位块/名"成为 O(1)（offset = 段起点 + index × 结构体大小）。
- **16**：Marshal 直接二进制读写零反射/无文本解析开销，且布局可控、紧凑、可随机定位。
- **17**：资源包打进归档，Resource 用 ReadFileSegment 按偏移读单个 AB 片段，避免海量小文件。
- **18**：非强加密；只为防止 strings/十六进制直接窥探资源清单。

</details>
