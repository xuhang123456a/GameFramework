## 参考答案要点

<details>
<summary>点开核对</summary>

- **1**：共享"组 + ObjectPool 实例复用 + 异步加载 + 生命周期钩子 + 迭代安全轮询"；UI 独有深度层级、暂停/遮挡、栈式焦点(Refocus)。
- **2**：OnPause/OnResume(暂停)、OnCover/OnReveal(遮挡)、OnRefocus(焦点)、OnDepthChanged(深度)。
- **3**：Depth=界面组整体深度；DepthInUIGroup=界面在组内的层叠序。
- **6**：Pause 停逻辑(OnUpdate/输入)，被会暂停的界面盖住时触发；Cover 仅视觉遮挡不停逻辑。
- **7**：每次 Open/Close/Refocus 后；从栈顶向下重算每个界面深度 + 暂停/遮挡状态并触发对应钩子。
- **8**：Refocus——移到栈顶 + OnRefocus，不新建第二个实例(单例界面)。
- **9**：OnResume(若之前被暂停) / OnReveal(若之前被遮挡) + 成为 CurrentUIForm。
- **10**：栈顶，深度最高(最上层)。
- **12**：界面间相互影响(暂停/遮挡是相对整个栈的)，局部更新易遗漏联动，导致状态不一致；全量刷新保证一致。
- **13**：界面卡在错误态——被暂停的界面上层关了却没 OnResume，永久停止响应。
- **14**：由"是否允许多开"参数/界面类型决定；一刀切单例会让需多开的界面(如多个物品详情)只能开一个；一刀切多开会让单例界面(如设置)叠出多份。
- **15**：抽出公共骨架(组+池+生命周期)，UI 只补栈层级；从头各写一遍重复维护对象池集成，易不一致。
- **16**：主界面 OnPause(被全屏暂停)；半透明提示在弹窗之上则正常，在其下则也被暂停。
- **17**：相同；复用实例(GameObject)，Close 时 Unspawn 回池，超容量/过期才 Release 销毁。
- **18**：深度/遮挡判断颠倒——栈顶界面被误判为被遮挡/暂停，栈底反而激活，界面层级全错。
- **21**：A=Paused(被 B 暂停)；B=Covered(被 C 遮挡，但 C 不暂停所以 B 不 Pause)；C=栈顶激活 Reveal。

</details>
